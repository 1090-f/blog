# 项目状态与后续建议

## 当前基线

项目已具备以下工程基础：

- Go 后端与 Vue 3 前端一体化构建和静态资源托管
- 公开服务与管理服务使用独立端口
- YAML 配置、`BLOG_*` 环境变量覆盖和 `BLOG_CONFIG_FILE`
- 用户认证、JWT 权限校验、管理员后台与用户状态控制
- 文章、分类、标签、评论、站点统计、归档活动和图片上传
- 后端测试、前端 lint/生产构建，以及 GitHub Actions CI

## 后续优先级

### P0：生产运维

- 补充实际部署环境的反向代理与 HTTPS 配置示例。
- 建立数据库和上传文件的备份、恢复与容量监控方案。
- 增加结构化日志采集和告警。

### P1：质量与安全

- 为认证、路由守卫、文章管理和上传流程补充前端自动化测试。
- 扩展后端接口的集成测试，覆盖权限边界和异常输入。
- 根据实际公开部署场景评估限流、审计日志和防滥用策略。

### P2：演进能力

- 制定显式的数据库版本迁移策略，逐步减少对长期 `AutoMigrate` 的依赖。
- 评估对象存储、CDN 与图片处理方案。
- 基于真实访问数据继续优化前端性能和内容检索体验。

## 维护约定

- 代码变更涉及配置、接口、启动方式或部署流程时，同步更新 README 和 `docs/`。
- 以实现代码、测试和 [API 文档](api.md) 为当前行为的最终依据；历史设计资料仅作参考。
